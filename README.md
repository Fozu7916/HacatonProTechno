# HacatonProTechno — Медиахаб для молодёжного центра

Сервис для медиакоманды молодёжного центра: создание контента, шаблоны, очереди, модерация, автопубликация в VK, аналитика и экспорт отчётов.

## Ключевые возможности

- Роли: `Волонтер`, `Редактор`, `СММ-специалист`, `Руководитель`.
- Генерация постов по шаблонам + ручное редактирование.
- ИИ-генерация текста (через OpenRouter/OpenAI-compatible endpoint).
- Ручной emoji-picker (выпадающее окно с категориями эмодзи, как в соцсетях).
- Вложения: фото, видео, документы (в т.ч. несколько файлов в одном посте).
- Очередь постов и календарь статусов (черновики / запланировано / опубликовано).
- Кастомный календарь с pop-up по клику на плашку в ячейке (просмотр/редактирование/удаление).
- Публикация в VK и защита от дублей публикации.
- Автопубликация по расписанию в фоне (`worker`).
- Дашборд руководителя + экспорт в `PDF` и `Excel` (включая графики).

## Технологии

- Python 3.11+
- Streamlit
- MySQL
- VK API
- pandas, matplotlib, fpdf2, xlsxwriter

## Быстрый старт

### 1) Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2) Подготовка базы данных

```sql
CREATE DATABASE vk_parser CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 3) Настройка `.env`

```env
GROUP_ID=123456789

DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=root
DB_NAME=vk_parser

# VK
VK_SERVICE_TOKEN=...
VK_USER_TOKEN=...

# LLM (опционально для ИИ-генерации)
OPENROUTER_API_KEY=...
LLM_BASE_URL=https://openrouter.ai/api/v1
LLM_MODEL=deepseek/deepseek-chat-v3-0324:free
```

### 4) Инициализация таблиц

В приложении есть кнопка `Инициализировать БД` в сайдбаре.

### 5) Запуск UI

```bash
python -m streamlit run app.py
```

## Автопубликация по расписанию (production-поток)

Запускается отдельным процессом:

```bash
python -m publisher.worker
```

Как работает:

- Воркер каждые 30 секунд проверяет очередь.
- Публикует только посты со статусом `ready`, у которых:
  - `scheduled_at <= NOW()` или
  - `scheduled_at IS NULL`.
- Один и тот же пост не может быть отправлен несколько раз (добавлена атомарная блокировка перед отправкой).
- Если публикация в VK не удалась, пост возвращается в `ready`.

Важно: для работы в реальном режиме `worker` должен быть постоянно запущен (в отдельном терминале/процессе).

## Работа с VK

- Кнопка `Обновить данные из VK` подтягивает посты в таблицу `posts`.
- Публикация постов выполняется модулем `publisher/scheduler.py`.
- Для вложений:
  - фото загружается как `photo...`,
  - видео/документы загружаются как `doc...`.

## Отчёты и экспорт

Для роли `Руководитель` доступен расширенный отчёт:

- KPI-блок,
- графики динамики,
- таблица по дням,
- топ-3 поста.

Экспорт:

- `PDF` — с KPI, графиками, таблицей и топом постов.
- `Excel` — данные + лист с графиками.

## CI/CD

Добавлены GitHub Actions:

- `CI` — `.github/workflows/ci.yml`
  - `push` в `main/master` и `pull_request`;
  - установка зависимостей;
  - проверка синтаксиса модулей;
  - smoke-check импорта `app.py`.

- `CD` — `.github/workflows/cd.yml`
  - `workflow_dispatch` и теги `v*`;
  - сборка zip-бандла проекта;
  - загрузка артефакта в GitHub Actions.

Примечание: текущий `CD` собирает артефакт, но не делает автодеплой на сервер.

## Структура проекта

```text
analytics/                  # аналитика и генерация PDF
database/                   # подключение к БД, SQL-операции
parser/                     # парсинг VK, загрузка вложений
publisher/                  # публикация и фоновой worker
custom_calendar_component/  # кастомный календарь с popup
app.py                      # основной Streamlit UI
```

## Ограничения на текущий момент

- Ролевая модель не 1-в-1 с ТЗ (`администратор/наблюдатель` пока не выделены отдельно).
- Нет кросс-постинга на несколько платформ.
- Нет конструктора опросов/викторин.

## Безопасность

- Не коммитьте `.env` в git.
- У `VK_USER_TOKEN` должны быть права на `wall` и операции с вложениями.
